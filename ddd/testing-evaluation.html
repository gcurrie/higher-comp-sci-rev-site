Testing a Database Solution

Testing a database involves verifying that:

  • The structure is correct — tables, keys and relationships are set up as designed
  • The validation rules work — invalid data is rejected; valid data is accepted
  • The SQL queries return the expected results
  • The CRUD operations (INSERT, UPDATE, DELETE) work correctly and maintain referential integrity

DDD Evaluation Criteria

⚠️

The DDD specification requires evaluation against two criteria only:

  1. Fitness for purpose
  2. Accuracy of output

Maintainability, robustness and usability are SDD evaluation criteria — do not use them for DDD evaluation answers.

1. Fitness for Purpose

A database solution is fit for purpose if it meets all the requirements identified during the analysis stage. This is assessed by comparing the implemented solution against the original functional requirements (what data needs to be stored, what queries need to be supported, what outputs are required).

What to evaluate

  • Are all required entities and their attributes stored?
  • Are the relationships between tables correct (matching the ER diagram)?
  • Are all the required queries implemented and returning data?
  • Do all CRUD operations work as expected?
  • Are the validation rules in place for the required fields?
Example Evaluation Comment — Fitness for Purpose

"The database stores all required entities (Member, Book, Loan) with their specified attributes and correctly links them using primary and foreign keys as shown in the ER diagram. All five required queries have been implemented and tested. The solution is fit for purpose."

2. Accuracy of Output

Accuracy of output means that the SQL queries return the correct, expected data — not just some data. The outputs must match what was specified in the requirements, with no missing, extra or incorrect records.

How to verify accuracy

  • Manually calculate what the correct output should be from known test data
  • Run the SQL query and compare actual output to expected output
  • Check that aggregate functions (COUNT, AVG, SUM) return mathematically correct values
  • Verify that WHERE conditions filter correctly — expected rows appear, excluded rows do not
  • Check that ORDER BY and GROUP BY produce the right ordering and groupings
Example Evaluation Comment — Accuracy of Output

"The query to find the average mark returned 72.5, which was manually verified against the test data (marks: 65, 78, 74, 73 → total 290 ÷ 4 = 72.5). The query to list pupils with marks ≥ 70 returned exactly the three pupils expected. All outputs are accurate."

Writing a DDD Evaluation

For each criterion, a good evaluation answer should:

  1. Name the criterion
  2. State what it means in a database context
  3. Provide specific evidence from your testing
  4. Reach a justified conclusion
Model Answer — Accuracy of Output (2 marks)

"The database output is accurate [1]. The query to count loans per member was tested against manually calculated expected values from the test data and returned the correct counts for all five test members [1]."
