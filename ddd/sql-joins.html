SQL — Joins

DDD Unit 2 Topic 5 of 7

A JOIN combines rows from two or more tables based on a related field (usually a primary key / foreign key pair). At Higher level, you need to use INNER JOIN.

Why Joins Are Needed

In a relational database, data is spread across multiple tables to avoid repetition. A join lets you bring that data back together in a query.

For example, a Pupil table stores TeacherID but not the teacher's name. To display a pupil's name alongside their teacher's name, you must join the Pupil and Teacher tables.

INNER JOIN Syntax

SELECT table1.field, table2.field
FROM table1
INNER JOIN table2
ON table1.primaryKey = table2.foreignKey;
💡
Use tableName.fieldName notation when the same field name appears in multiple tables to avoid ambiguity.

Basic INNER JOIN Example

-- Display each pupil's name alongside their teacher's name
SELECT Pupil.FirstName, Pupil.LastName,
       Teacher.FirstName AS TeacherFirst,
       Teacher.LastName AS TeacherLast
FROM Pupil
INNER JOIN Teacher
ON Pupil.TeacherID = Teacher.TeacherID;

JOIN with WHERE and ORDER BY

-- Pupils in the Computing department, sorted by score
SELECT Pupil.FirstName, Pupil.LastName, Pupil.Score,
       Teacher.Department
FROM Pupil
INNER JOIN Teacher
ON Pupil.TeacherID = Teacher.TeacherID
WHERE Teacher.Department = 'Computing'
ORDER BY Pupil.Score DESC;

Joining Three Tables

You can chain multiple JOINs to combine three or more tables:

-- Example: Show pupil name, club name and date joined
SELECT Pupil.FirstName, Pupil.LastName,
       Club.ClubName, PupilClub.DateJoined
FROM Pupil
INNER JOIN PupilClub ON Pupil.PupilID = PupilClub.PupilID
INNER JOIN Club ON PupilClub.ClubID = Club.ClubID
ORDER BY Club.ClubName;
