✅Revision Checklists

💻 Software Design & Development
  • ☐ Languages & Environments (compilers, interpreters, IDEs)
  • ☐ Data types (integer, real, boolean, character, string)
  • ☐ Data structures (1D array, 2D array, record)
  • ☐ Computational constructs (loops, selection, subprograms)
  • ☐ Parameter passing (by value, by reference)
  • ☐ Standard algorithms (input validation, find min/max, linear search, count)
  • ☐ Testing (error types, test data types, trace tables)
  • ☐ Design notations (structure diagrams, pseudocode, flowcharts)
  • ☐ Evaluation (fitness for purpose, efficiency, maintainability, readability)
🗄️ Database Design & Development
  • ☐ Entities, attributes, relationships
  • ☐ Relationship types (1:1, 1:M, M:M)
  • ☐ ER diagrams and cardinality notation
  • ☐ Primary keys and foreign keys
  • ☐ Referential integrity
  • ☐ SQL SELECT with WHERE and ORDER BY
  • ☐ Aggregate functions (COUNT, SUM, AVG, MAX, MIN)
  • ☐ SQL INNER JOIN
  • ☐ SQL INSERT, UPDATE, DELETE
🖥️ Computer Systems
  • ☐ Binary, denary and hexadecimal conversion
  • ☐ Two's complement (negative numbers)
  • ☐ Floating-point (mantissa and exponent)
  • ☐ ASCII and Unicode
  • ☐ Bitmap graphics (resolution, colour depth, file size)
  • ☐ Vector graphics (comparison with bitmap)
  • ☐ Sound (sampling rate, bit depth, file size)
  • ☐ CPU (ALU, CU, registers), memory (RAM, ROM, cache), buses
  • ☐ Fetch-execute cycle
  • ☐ Environmental impact and green computing
  • ☐ Security threats and precautions

🎯General Exam Tips

📖
Read the question carefully. Note exactly how many marks are available — each mark typically requires a distinct point.
💬
Use technical language. Use correct computing terminology — "compiler", "iteration", "primary key" — rather than vague descriptions.
🧮
Show your working. For calculation questions (file size, binary conversion), always show each step — you can gain marks for method even if the final answer is wrong.
⏱️
Time management. The paper is 2 hours for 80 marks — roughly 1.5 minutes per mark. Don't spend too long on any single question.
🔄
Apply to context. When evaluating, always refer back to the specific scenario in the question rather than giving generic answers.
✅
Check SQL syntax. In SQL questions, check capitalisation of keywords, correct use of quotes around text values, and WHERE clause conditions.
