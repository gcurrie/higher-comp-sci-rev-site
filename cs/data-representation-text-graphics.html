Extended ASCII

ASCII (American Standard Code for Information Interchange) was the original standard for encoding text characters as binary numbers. The original 7-bit ASCII could represent 128 characters (0–127): uppercase and lowercase English letters, digits, punctuation, and control codes.

Extended ASCII uses 8 bits per character, allowing 256 characters (0–255). The extra 128 values were used for accented characters, box-drawing symbols and other regional characters — but different manufacturers defined these extra characters differently, causing inconsistency.

Unicode

Unicode is a universal character encoding standard designed to represent every character from every writing system in the world. It assigns a unique code point to over 140,000 characters, covering Latin, Arabic, Chinese, Japanese, Korean, Greek, Cyrillic, emoji, and many more.

The most common Unicode encoding is UTF-8:

  • Variable-length: characters use 1–4 bytes depending on their code point
  • The first 128 characters (standard ASCII) are encoded identically in ASCII and UTF-8 — backwards compatible
  • Used on over 98% of all websites

UTF-16 uses 2 or 4 bytes per character. Used internally by Windows and Java.

The key difference the exam tests: ASCII uses 8 bits (256 characters, Western European only); Unicode supports every writing system using variable-length encoding but requires more storage for non-Latin characters.

Bitmap Graphics

A bitmap image (also called a raster image) stores the colour of every individual pixel in a grid. Common formats: JPEG, PNG, GIF, BMP.

Key Terms

TermDefinition
ResolutionThe number of pixels in the image, usually stated as width × height (e.g. 1920×1080). More pixels = more detail = larger file.
Colour depthThe number of bits used to represent each pixel's colour. 8-bit = 256 colours; 24-bit = 16.7 million colours (true colour).
File sizeResolution × colour depth (in bits), then convert to bytes. A 100×100 pixel image at 24-bit colour = 100 × 100 × 24 = 240,000 bits = 30,000 bytes ≈ 29 KB (uncompressed).
File Size Calculation

An image is 800 × 600 pixels with a colour depth of 24 bits.
File size = 800 × 600 × 24 = 11,520,000 bits = 1,440,000 bytes = 1.37 MB (uncompressed)

Advantages of Bitmap

  • Can represent complex, detailed images with many colours (photographs)
  • Widely supported by all image editing and display software
  • Easy to edit individual pixels

Disadvantages of Bitmap

  • Large file sizes for high-resolution or high-colour-depth images
  • Becomes pixelated (blurry) when scaled up — resolution is fixed
  • Not suitable for logos or line art that need to scale to any size

Vector Graphics

A vector image stores the image as a collection of mathematical descriptions of shapes — lines, curves, polygons — defined by coordinates, colours and properties. Common formats: SVG, AI, EPS.

Advantages of Vector

  • Can be scaled to any size without loss of quality — the shapes are recalculated for the new size
  • Smaller file sizes for simple images (logos, diagrams, line art)
  • Easy to edit individual shapes, colours and elements
  • Ideal for print design, logos and illustrations

Disadvantages of Vector

  • Not suitable for complex photographic images — storing a photograph as mathematical shapes is impractical
  • Rendering complex vector images can be slow
  • Less widely supported than bitmap formats
