Binary Integers (Unsigned)

Computers store all data in binary — using only the digits 0 and 1. Each binary digit is called a bit.

An 8-bit (1 byte) number can represent unsigned values from 0 to 255 (2⁸ − 1).

Two's Complement

Two's complement is the method used to represent both positive and negative integers in binary. It uses the most significant bit (MSB — the leftmost bit) as a sign bit, but with a twist: the MSB has a negative place value.

Place Values in Two's Complement (8-bit)

−1286432168421
MSB (sign) → these remain positive as usual
  • If MSB = 0, the number is positive
  • If MSB = 1, the number is negative (the MSB contributes −128)

Converting Two's Complement to Denary

Example 1 — Positive: 01001010

MSB = 0, so positive.
0×(−128) + 1×64 + 0×32 + 0×16 + 1×8 + 0×4 + 1×2 + 0×1 = 74

Example 2 — Negative: 11110110

MSB = 1, so negative.
1×(−128) + 1×64 + 1×32 + 1×16 + 0×8 + 1×4 + 1×2 + 0×1
= −128 + 64 + 32 + 16 + 0 + 4 + 2 + 0 = −10

Converting a Negative Denary to Two's Complement

To represent a negative number (e.g. −45) in 8-bit two's complement:

  1. Write the positive version in binary: 45 = 00101101
  2. Flip all bits (invert): → 11010010
  3. Add 1: → 11010011

Check: 1×(−128) + 1×64 + 1×16 + 0×8 + 0×4 + 1×2 + 1×1 = −128 + 64 + 16 + 2 + 1 = −45 ✓

💡
An 8-bit two's complement number can represent values from −128 to +127 (2⁷ negative to 2⁷ − 1 positive). Notice it is not symmetric — there is one more negative value than positive.

Floating-Point Representation

Floating-point is used to represent real numbers (numbers with a fractional part, including very large or very small values) in binary. It can represent both positive and negative real numbers.

The name "floating-point" comes from the fact that the binary point can "float" — it can be positioned anywhere, allowing the representation of a wide range of values.

Why Not Just Fixed-Point?

A fixed-point format assigns a fixed number of bits to the integer part and to the fractional part. This limits the range. Floating-point uses a compact scientific-notation-style encoding to represent a much wider range of values in the same number of bits.

Mantissa and Exponent

A floating-point number is split into two parts:

PartWhat it isAnalogy
Mantissa The significant digits of the number (the "precision") The 3.14 in 3.14 × 10²
Exponent The power by which the mantissa is scaled (where the binary point goes) The 2 in 3.14 × 10²

In binary: a floating-point number is stored as mantissa × 2exponent.

Example — 8-bit floating-point (5-bit mantissa, 3-bit exponent)

Representation: 0 1 1 0 1 | 0 1 1

Mantissa: 01101 = the value 0.1101 in binary (normalised — binary point after first bit)
Exponent: 011 = +3 (shift binary point 3 places right)

Value: 0.1101 × 2³ = 1101.0 = 8 + 4 + 1 = 13

Effect of Changing Bit Allocation

ChangeEffect on mantissa bitsEffect on exponent bits
More bits to mantissaGreater precision (more decimal places accurate)Smaller range (fewer different magnitudes possible)
More bits to exponentLower precision (fewer accurate decimal places)Greater range (can represent larger/smaller values)
ℹ️
Key terms for exam answers:
  • Precision — how many significant digits are stored accurately (more mantissa bits = more precise)
  • Range — how large or small a number can be (more exponent bits = wider range)
