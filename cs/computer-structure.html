Key Hardware Components

ComponentRole
CPU (Central Processing Unit) Carries out all processing — fetches, decodes and executes instructions
ALU (Arithmetic Logic Unit) Part of the CPU; performs arithmetic (add, subtract) and logical (AND, OR, compare) operations
Control Unit (CU) Part of the CPU; manages the fetch-execute cycle — directs data flow between CPU, memory and devices
Registers Tiny, extremely fast storage locations inside the CPU. Key registers: Program Counter (PC), Memory Address Register (MAR), Memory Data Register (MDR), Accumulator (ACC)
Cache Memory Small, very fast memory between the CPU and RAM. Stores frequently used instructions and data to speed up access
RAM (Random Access Memory) Main working memory — stores programs and data currently in use. Volatile (lost when power off)
ROM (Read-Only Memory) Stores permanent startup instructions (BIOS/firmware). Non-volatile
Address Bus Carries memory addresses from CPU to RAM
Data Bus Carries data between CPU and RAM (in both directions)
Control Bus Carries control signals (read/write commands, clock signals)

Fetch-Execute Cycle

The fetch-execute cycle (also called the instruction cycle) is the continuous process by which the CPU retrieves and carries out program instructions.

1
Fetch

PC holds address of next instruction. MAR copies address. Instruction loaded from RAM into MDR.

2
Decode

Control unit interprets the instruction in MDR — determines what operation is required.

3
Execute

ALU performs the required operation. PC increments to point to next instruction. Cycle repeats.

Registers in the Fetch Stage

RegisterFull nameRole in fetch cycle
PCProgram CounterHolds the address of the next instruction to fetch; increments after each fetch
MARMemory Address RegisterReceives the address from PC; holds the address being accessed on the bus
MDRMemory Data RegisterTemporarily holds the instruction or data fetched from RAM
ACCAccumulatorStores the result of ALU operations during execute stage

Factors Affecting Computer Performance

The Higher CS specification identifies four specific factors that affect how fast a computer can process instructions:

🔲

Number of Processors (Cores)

A multi-core processor has multiple processing units on one chip. Each core can execute a different instruction stream simultaneously.

Effect: More cores = more instructions processed simultaneously = better performance on multi-threaded workloads.

🚌

Width of the Data Bus

The data bus carries data between the CPU and memory. A wider bus (more parallel wires) can carry more bits in a single transfer.

Effect: A 64-bit data bus transfers twice as much data per cycle as a 32-bit bus — reduces the number of transfers needed for large data values.

⚡

Cache Memory

Cache stores frequently used data and instructions between the CPU and slower main RAM. The CPU checks cache before fetching from RAM.

Effect: More / larger cache = fewer slow RAM accesses = faster execution. Cache access is 10–100× faster than RAM.

⏱️

Clock Speed

The system clock generates regular pulses that synchronise CPU operations. One fetch-execute cycle takes a fixed number of clock pulses.

Effect: Higher clock speed (measured in GHz) = more cycles per second = more instructions executed per second.

💡
Exam questions often ask you to explain how increasing a factor improves performance. Always state what changes and why that speeds up processing — don't just say "it makes it faster."
Model Answer — Clock Speed (2 marks)

"Increasing the clock speed means the CPU completes more clock cycles per second [1]. Since each instruction is executed in a fixed number of clock cycles, more cycles per second means more instructions are executed per second, so the processor runs faster [1]."
