Types of Programming Language

Programming languages can be grouped into levels based on how close they are to the hardware:

LevelExampleDescription
Machine codeBinary (0s and 1s)Directly executed by the processor. Extremely fast but very difficult to read or write.
Assembly languageMOV, ADD, JMPUses mnemonics instead of binary. Still hardware-specific; converted by an assembler.
High-level languagePython, Java, C#Human-readable syntax. Must be translated before the processor can run it.
Scripting languageJavaScript, PythonTypically interpreted, often used for automating tasks or web interactivity.

Translators

A translator converts source code (written by a programmer) into machine code (understood by the processor). There are three types:

Compiler
Translates the entire source code into machine code in one go, producing a separate executable file. The original source code is not needed to run the program. Errors are reported after full compilation.
Interpreter
Translates and executes source code line-by-line at runtime. The source code must be present each time the program runs. Errors are reported as they are encountered during execution.
Assembler
Translates assembly language (mnemonics) into machine code. Used for low-level, hardware-specific programming.

Compiler vs Interpreter — Comparison

FeatureCompilerInterpreter
TranslationWhole program at onceLine by line at runtime
SpeedFaster execution (pre-compiled)Slower (translates each run)
Error reportingAfter full compilationAt the first error encountered
Source code needed to runNoYes
ExamplesC, C++, Java (to bytecode)Python, JavaScript

Integrated Development Environments (IDEs)

An IDE is a software application that provides all the tools a programmer needs in one place. Key features include:

  • Code editor — with syntax highlighting and auto-completion
  • Translator — built-in compiler or interpreter
  • Debugger — allows stepping through code, setting breakpoints, inspecting variables
  • Error highlighting — marks syntax errors as you type
  • Run environment — execute programs directly within the IDE

Examples of IDEs: Visual Studio Code, PyCharm, IDLE (Python), Replit.

Key Terms

Source code
The human-readable instructions written by a programmer in a high-level language.
Machine code
Binary instructions (0s and 1s) that the processor can execute directly.
Executable
A file containing compiled machine code that can be run without the original source code.
Syntax
The rules that govern the structure of valid statements in a programming language.
Mnemonic
A short, memorable code used in assembly language (e.g. MOV, ADD) to represent a machine code instruction.
