Data Structures

At Higher level you work with three data structures: parallel 1D arrays, records, and arrays of records.

On this page

ℹ️
Higher vs National 5: At National 5 you used simple 1D arrays. At Higher the three required structures are parallel 1D arrays, records, and arrays of records. 2D arrays are not a required data structure at Higher.

Parallel 1D Arrays

Two or more separate 1D arrays where the same index position holds related information about the same item. The arrays must always have the same length and be maintained together.

SQA Reference Language — Parallel Arrays
DECLARE names  AS ARRAY OF STRING INITIALLY []
DECLARE marks  AS ARRAY OF INTEGER INITIALLY []
DECLARE grades AS ARRAY OF STRING INITIALLY []

# names[0], marks[0] and grades[0] all describe the same pupil
SET names[0]  TO "Alice"
SET marks[0]  TO 78
SET grades[0] TO "B"

SET names[1]  TO "Bob"
SET marks[1]  TO 92
SET grades[1] TO "A"
💡
Parallel arrays are useful when you want to apply standard algorithms (find max, count occurrences) on a single numeric array, then use the same index to look up related data from a second array.

Using the Index to Link Related Data

To display the name of the pupil with the highest mark, find the index of the maximum in marks, then use that index in names:

DECLARE maxIndex AS INTEGER INITIALLY 0
FOR i FROM 1 TO LENGTH(marks) - 1 DO
    IF marks[i] > marks[maxIndex] THEN
        SET maxIndex TO i
    END IF
END FOR
SEND "Top pupil: " & names[maxIndex] & " scored " & marks[maxIndex] TO DISPLAY

Records

A record is a collection of named fields that can each hold a different data type. Where an array stores many values of the same type, a record stores multiple attributes of a single item.

Defining and Using a Record
RECORD Pupil IS
    name   : STRING
    mark   : INTEGER
    passed : BOOLEAN
END RECORD

DECLARE student AS Pupil

SET student.name   TO "Claire"
SET student.mark   TO 65
SET student.passed TO TRUE

SEND student.name & " scored " & student.mark TO DISPLAY
FeatureArrayRecord
StoresMany values, same typeMultiple fields, different types
AccessBy numeric index: arr[2]By field name: student.name
Use caseA list of the same kind of valueA single complex item with named attributes

Arrays of Records

An array of records combines both structures: a 1D array where every element is a record. This allows you to store a list of complex items, each with multiple named fields of different types.

Declaring an Array of Records
RECORD Pupil IS
    name : STRING
    mark : INTEGER
END RECORD

DECLARE class AS ARRAY OF Pupil INITIALLY []

SET class[0].name TO "Alice"
SET class[0].mark TO 78

SET class[1].name TO "Bob"
SET class[1].mark TO 92

Iterating Over an Array of Records

FOR i FROM 0 TO LENGTH(class) - 1 DO
    SEND class[i].name & ": " & class[i].mark TO DISPLAY
END FOR

Comparison: Parallel Arrays vs Array of Records

Parallel ArraysArray of Records
Related data kept together?Only by convention (same index)Yes — one element holds all fields
Risk of mismatchHigher — must keep arrays in syncNone — fields are part of the same element
Field accessIndex: names[i], marks[i]Dot notation: class[i].name
Preferred forSimple numeric algorithms on one attributeStoring and processing structured records

Choosing a Data Structure

  • Use a 1D array for a simple list of single values of the same type.
  • Use parallel 1D arrays when you have related data of different types and want standard algorithms on one numeric array.
  • Use a record for a single item with named attributes of different types.
  • Use an array of records for a list of items that each have multiple attributes of different types — the most common structure in Higher assignments.
⚠️
Answers mentioning "2D arrays" for Higher data structures will not gain marks. The required structures are parallel arrays, records, and arrays of records.
