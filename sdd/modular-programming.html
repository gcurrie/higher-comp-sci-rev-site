Functions and Procedures

Breaking a program into subprograms (also called modules) is central to Higher-level programming. It makes code easier to test, maintain and reuse.

ProcedureFunction
Returns a value?NoYes — one value via RETURN
Used forPerforming an action (e.g. display output)Calculating and returning a result
Called byStatement: displayMenu()Expression: SET total TO add(a, b)
Procedure and Function Examples
# Procedure — performs an action, no return value
PROCEDURE displayGreeting(name : STRING)
    SEND "Hello, " & name TO DISPLAY
END PROCEDURE

# Function — calculates and returns a value
FUNCTION square(n : INTEGER) RETURNS INTEGER
    RETURN n * n
END FUNCTION

# Calling them
CALL displayGreeting("Alice")
SET result TO square(5)    # result = 25

Parameter Passing

Parameters allow data to be passed into a subprogram. There are two terms to know:

TermWhere?Definition
Formal parameter In the subprogram definition The named placeholder that receives the value
Actual parameter In the subprogram call The real value or variable passed in when calling
Formal vs Actual Parameters
FUNCTION add(a : INTEGER, b : INTEGER) RETURNS INTEGER
#              ^ formal      ^ formal
    RETURN a + b
END FUNCTION

SET total TO add(10, 20)
#                ^ actual  ^ actual
💡
Exam questions often ask you to "identify the formal parameters" or "state the actual parameters used in the call." Always check where you are looking — definition = formal, call = actual.

Scope: Local and Global Variables

The scope of a variable determines where in a program it can be accessed.

Local VariableGlobal Variable
Declared insideA subprogramThe main program (outside all subprograms)
AccessibleOnly within that subprogramThroughout the entire program
ExistsOnly while subprogram runsFor the lifetime of the program
Preferred?Yes — reduces unintended side effectsAvoid where possible
 
DECLARE total AS INTEGER INITIALLY 0   # global

PROCEDURE addToTotal(amount : INTEGER)
    DECLARE temp AS INTEGER INITIALLY 0  # local to this procedure
    SET temp TO amount * 2
    SET total TO total + temp
END PROCEDURE

Pre-defined Functions

The SQA Higher exam expects you to know and use four categories of pre-defined functions. These are built-in operations provided by the language.

1. Substrings

Extract part of a string using a start position and a length.

DECLARE word AS STRING INITIALLY "Computing"

SET part TO SUBSTRING(word, 0, 4)   # Returns "Comp"
# SUBSTRING(string, start index, length)
Watch

2. Character ↔ ASCII (chr and ord)

Convert between a character and its ASCII (Unicode) code number.

SET code TO ASC("A")      # Returns 65
SET letter TO CHR(65)     # Returns "A"

# Useful for checking alphabetical order, shifting characters
IF ASC(char) >= ASC("a") AND ASC(char) <= ASC("z") THEN
    SEND "Lowercase letter" TO DISPLAY
END IF

3. Real to Integer

Convert a floating-point (real) number to an integer by truncating (removing) the decimal part.

DECLARE price AS REAL INITIALLY 9.99

SET whole TO INT(price)    # Returns 9 (truncates, does not round)
SET rounded TO ROUND(price, 0)  # Returns 10.0

4. Modulus (Remainder)

The modulus operator returns the remainder after integer division. It is used to check divisibility, detect even/odd numbers, and cycle through indices.

SET remainder TO 17 MOD 5    # Returns 2  (17 = 3×5 + 2)
SET isEven   TO n MOD 2 = 0  # TRUE if n is even

# Check if a number is divisible by 3:
IF n MOD 3 = 0 THEN
    SEND n & " is divisible by 3" TO DISPLAY
END IF

Selection and Iteration

These constructs carry over from National 5 and are used throughout Higher programming tasks.

Selection

# Simple IF
IF mark >= 70 THEN
    SET grade TO "A"
ELSE IF mark >= 60 THEN
    SET grade TO "B"
ELSE
    SET grade TO "C"
END IF

# CASE (equivalent to switch/match)
CASE grade OF
    "A" : SEND "Distinction" TO DISPLAY
    "B" : SEND "Merit" TO DISPLAY
    OTHERWISE : SEND "Pass" TO DISPLAY
END CASE

Iteration

# Fixed loop
FOR i FROM 0 TO 9 DO
    SEND class[i].name TO DISPLAY
END FOR

# Conditional loop (pre-test)
WHILE score < 0 OR score > 100 DO
    RECEIVE score FROM KEYBOARD
END WHILE

# Conditional loop (post-test — runs at least once)
REPEAT
    RECEIVE score FROM KEYBOARD
UNTIL score >= 0 AND score <= 100
