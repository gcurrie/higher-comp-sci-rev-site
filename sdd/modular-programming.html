Modular Programming

SDD Unit 1 Topic 8 of 10

Modular programming breaks a program down into smaller, reusable sub-programs called modules, making code easier to write, test and maintain. This section covers functions, procedures and parameters, the scope of variables, and how to combine modules into a fully modular program.

Functions, Procedures & Parameters

Modular programming means breaking a program down into smaller sub-programs (also called modules). In Python there are two types of module: functions and procedures. Both are defined using the def keyword.

Functions

A function is a sub-program that returns a value, so the final line of the function should be a return statement.

Python — Basic Function Structure
def get_speed(distance, time):
  speed = distance / time
  return speed

Calling myTopSpeed = get_speed(110, 35) would return approximately 3.14 and store it in myTopSpeed.

Python — Function to Square a Number
# Function to square a number
def square(n):
  s = n * n
  return s

# Main part of the program...
number = int(input("Please enter a number"))
sq = square(number)
print(sq)
Python — Function Using the Find Max Algorithm
# A function using the find max algorithm
def findmax(array_of_nums):
  max = array_of_nums[0]
  for index in range(1, len(array_of_nums)):
    if array_of_nums[index] > max:
      max = array_of_nums[index]
  return max

# Main program...
scores = [200, 55, 84, 21, 985, 300]
high_score = findmax(scores)
print(high_score)

Procedures

A procedure is a sub-program that does not return a value. The key difference is the absence of a return statement. Procedures are used when data only needs to flow into the module (for example, to display output).

Python — Procedure to Add and Print
def add_and_print(num1, num2):
  total = num1 + num2
  print("The total is", total)

# Main program
number1 = int(input("Please enter number 1"))
number2 = int(input("Please enter number 2"))
add_and_print(number1, number2)

Returns a value?NoYes — one value via return
Ends withNo return statementA return statement
Used forPerforming an action (e.g. display output)Calculating and returning a result

Parameters

A parameter is a piece of data that is passed into a function or procedure. This can be a static piece of data hardcoded into the program, or a variable. There are two types of parameter: actual and formal.

TypeWhere?Definition
Formal parameter In the def statement (where the module is defined) The named placeholder that receives the value
Actual parameter In the brackets when the module is called (where it is used) The real value or variable being passed in
Python — Formal Parameters (in the definition)
def get_speed(distance, time):
#             ^ formal   ^ formal
  speed = distance / time
  return speed
Python — Actual Parameters (in the call)
high_score = findmax(scores)
#                    ^ actual
total = add_two_numbers(num1, num2)
#                       ^ actual ^ actual
display_total(total)
#             ^ actual
💡
Exam questions (for example, 2019) often ask you to identify one formal and one actual parameter from a piece of code. Always check where you are looking — parameters in the def statement are formal; parameters in the brackets when the module is called are actual.

Scope of Variables

The scope of a variable determines where in a program it can be used. Variables are either global or local.

Global Variables

Global variables can be used anywhere in the program. They are declared in the main program, outside any function block (with no indentation).

Python — Global Variables
username = "admin"
scores = [22, 5, 10, 18, 9]

Global variables should not be over-used — they stay in memory for longer than necessary, which is less efficient and increases the risk of errors.

Local Variables

Local variables can only be used in one section of code (for example, in a single function or procedure). In the example below, total is local to the function and cannot be accessed outside it.

Python — Local Variables
def myfunction(param1, param2):
  total = param1 + param2
  return total

number1 = 5
number2 = myfunction(number1, 6)
✅
Advantages of local variables:
  • They are released from memory when no longer needed, so they free up resources.
  • The same name can be re-used in different parts of the program without the variables interfering with each other.
  • You cannot accidentally affect another part of the program, which avoids logic errors.

Making a Global Variable Accessible

Using the global keyword allows a function to reference and change a program-level (global) variable.

Python — The global Keyword
total = 0

def myfunction(param1, param2):
  global total
  total = param1 + param2
  return total
Python — Identifying Local and Global Variables
# Find Max Algorithm
def findMax(list):
  max = list[0]
  for counter in range(len(list)):
    if list[counter] > max:
      max = list[counter]
  return max

# Declare an array of floats to store a list of BMIs
bmis = [22.3, 26.3, 33.22, 21.22, 17.23]
heaviest = findMax(bmis)

Here max and list are local (function-scoped), while bmis and heaviest are global (declared in the main program).

Tying it All Together

Functions and procedures are combined to build a complete program. Remember:

  • A function returns a value (data going out).
  • A procedure is like a function, but without a return statement.
  • Both functions and procedures can accept passed parameters.

A good approach is to begin by writing the module stubs (just the def lines) before filling in the detail of each one.

Python — Module Stubs
# Read song data from text file
def read_file():

# Count number of downloads for each song
def count_downloads(songdata):

# Find song with most downloads
def find_most_dl(howmany):

# Display song info
def display_info(mostpopular, songdata):

Once the modules are defined, add the main algorithm at the end of the file. This calls each module in turn, passing data between them.

Python — Main Algorithm
# Main algorithm
s = read_file()
h = count_downloads(s)
m = find_most_dl(h)
display_info(m, s)
ℹ️
The order you write the modules in the file doesn't matter, so long as they all come before the main algorithm.

Complete Working Program

Python — Fully Modular Program
def get_countries():
  file = open("countries.txt", "r")
  countries = file.readlines()
  file.close()
  return countries

def get_letter():
  letter = input("Please enter a capital letter:")
  while len(letter) != 1 or (ord(letter) < 65 or ord(letter) > 90):
    print("Error: input is not valid")
    letter = input("Please enter a capital letter:")
  return letter

def count_countries(countries, letter):
  count = 0
  for index in range(0, len(countries)):
    first_letter = countries[index][0:1]
    if letter == first_letter:
      count = count + 1
  return count

def show_result(count):
  print("Matches found:", count)

c = get_countries()
l = get_letter()
count = count_countries(c, l)
show_result(count)

Benefits of Modular Programming

Breaking a program into modules has a number of important advantages:

  • Team working — multiple developers can work on different modules (parts of the project) at the same time.
  • Independent testing — each module can be tested on its own, which makes errors far easier to locate and fix.
  • Efficient use of memory — local variables are only held in memory while their module is running, then released, so resources are freed up.
  • Fewer errors from variable scope — because modules mainly use local variables, one module is far less likely to accidentally change data belonging to another.
  • Reusability — a well-written module can be reused in other programs without being rewritten.
  • Readability and maintenance — each module does one clear job, so the program is easier to read, understand and update later.
