Standard Algorithms

At Higher level, three standard algorithms are required: linear search, find minimum and maximum, and count occurrences. These can be applied to 1D arrays or arrays of records.

Which Algorithms Are Required at Higher?

⚠️

Higher requires only three standard algorithms:

  • Linear search
  • Find minimum and maximum
  • Count occurrences

Input validation and running total are National 5 algorithms — they will not be asked in a Higher exam context as a "standard algorithm" question.

Linear Search

A linear search checks every element in an array one at a time, from the first element to the last, until the target value is found or the end of the array is reached.

Key Steps

  1. Set a flag (found) to FALSE and an index to 0
  2. Loop through each element
  3. If the element matches the target, set found to TRUE and record the index
  4. After the loop, report whether the target was found
Linear Search — 1D Array
DECLARE names AS ARRAY OF STRING INITIALLY ["Alice","Bob","Claire","David"]
DECLARE target AS STRING INITIALLY "Claire"
DECLARE found AS BOOLEAN INITIALLY FALSE
DECLARE foundAt AS INTEGER INITIALLY -1

FOR i FROM 0 TO LENGTH(names) - 1 DO
    IF names[i] = target THEN
        SET found TO TRUE
        SET foundAt TO i
    END IF
END FOR

IF found THEN
    SEND target & " found at index " & foundAt TO DISPLAY
ELSE
    SEND target & " not found" TO DISPLAY
END IF
💡
Some exam mark schemes accept a version that exits the loop early once found (using a WHILE NOT found loop). Either approach is valid — check what your teacher prefers for coursework.

Variation: Exit Early When Found

DECLARE i AS INTEGER INITIALLY 0
DECLARE found AS BOOLEAN INITIALLY FALSE

WHILE i < LENGTH(names) AND NOT found DO
    IF names[i] = target THEN
        SET found TO TRUE
    ELSE
        SET i TO i + 1
    END IF
END WHILE

Find Minimum and Maximum

Scan through the entire array, keeping track of the smallest (or largest) value seen so far. After the loop, the tracked value is the minimum (or maximum).

Finding the Minimum

Find Minimum Value and Its Index
DECLARE marks AS ARRAY OF INTEGER INITIALLY [78, 45, 92, 61, 55]
DECLARE minIndex AS INTEGER INITIALLY 0

FOR i FROM 1 TO LENGTH(marks) - 1 DO
    IF marks[i] < marks[minIndex] THEN
        SET minIndex TO i
    END IF
END FOR

SEND "Lowest mark: " & marks[minIndex] & " (index " & minIndex & ")" TO DISPLAY

Finding the Maximum

DECLARE maxIndex AS INTEGER INITIALLY 0

FOR i FROM 1 TO LENGTH(marks) - 1 DO
    IF marks[i] > marks[maxIndex] THEN
        SET maxIndex TO i
    END IF
END FOR

SEND "Highest mark: " & marks[maxIndex] TO DISPLAY
ℹ️
Always start from index 1 (not 0) in the loop — you use index 0 as the initial best candidate, then compare all others against it. Starting the loop from 0 would make an unnecessary comparison of the first element with itself.

Count Occurrences

Traverse the entire array, incrementing a counter each time the element matches the target value.

Count Occurrences of a Value
DECLARE grades AS ARRAY OF STRING INITIALLY ["A","B","A","C","A","B"]
DECLARE target AS STRING INITIALLY "A"
DECLARE count AS INTEGER INITIALLY 0

FOR i FROM 0 TO LENGTH(grades) - 1 DO
    IF grades[i] = target THEN
        SET count TO count + 1
    END IF
END FOR

SEND "Number of " & target & " grades: " & count TO DISPLAY

Applying Algorithms to Arrays of Records

All three algorithms work with arrays of records — you simply specify which field to compare.

Find Maximum in an Array of Records
RECORD Pupil IS
    name : STRING
    mark : INTEGER
END RECORD

DECLARE class AS ARRAY OF Pupil INITIALLY []
# (assume class is populated with pupil data)

DECLARE maxIndex AS INTEGER INITIALLY 0

FOR i FROM 1 TO LENGTH(class) - 1 DO
    IF class[i].mark > class[maxIndex].mark THEN
        SET maxIndex TO i
    END IF
END FOR

SEND "Top pupil: " & class[maxIndex].name &
     " with " & class[maxIndex].mark TO DISPLAY
