File Handling

At Higher level you are expected to read data from and write data to files using sequential access. The two most common formats are plain text files (.txt) and comma-separated value files (.csv).

ℹ️
Key file-handling terms: OPEN (open a file for reading or writing), CLOSE (release the file when done), EOF (end of file — used to detect when all lines have been read), sequential access (reading from start to finish in order).

Reading from a File

Open the file for reading, then loop until the end of file (EOF) is reached, receiving one line at a time.

Reading all lines from a text file
OPEN "scores.txt" FOR READING AS myFile

WHILE NOT EOF(myFile) DO
    RECEIVE line FROM myFile
    SEND line TO DISPLAY
END WHILE

CLOSE myFile

Writing to a File

Open the file for writing, then send each piece of data to the file. Remember to close the file when finished so the data is saved.

Writing lines to a text file
OPEN "output.txt" FOR WRITING AS myFile

FOR i FROM 0 TO LENGTH(names) - 1 DO
    SEND names[i] & "," & marks[i] TO myFile
END FOR

CLOSE myFile

Reading a CSV File into an Array of Records

A common Higher task is to read a CSV file line by line, splitting each line into fields and storing them in an array of records.

OPEN "pupils.csv" FOR READING AS csvFile

DECLARE i AS INTEGER INITIALLY 0
WHILE NOT EOF(csvFile) DO
    RECEIVE line FROM csvFile
    SET class[i].name TO FIELD(line, ",", 0)   # first field
    SET class[i].mark TO INT(FIELD(line, ",", 1))  # second field
    SET i TO i + 1
END WHILE

CLOSE csvFile
