Development Methodologies

SDD Unit 1 Topic 8 of 9

A software development methodology is a structured approach to planning, creating and managing a software project. Different methodologies suit different project sizes, teams and requirements.

The Software Development Process

Regardless of methodology, all software projects follow these stages:

  1. Analysis — understand the problem and gather requirements
  2. Design — plan the solution (structure diagrams, pseudocode, wireframes)
  3. Implementation — write the code
  4. Testing — verify the program works correctly
  5. Evaluation — assess the finished product against requirements
  6. Maintenance — fix bugs and update the software after release

Iterative Development

In iterative development, the software is built and refined through repeated cycles (iterations). Each iteration produces a working version of part of the software, which is reviewed and improved. This is the approach emphasised at Higher level.

  • Work is done in small, manageable cycles
  • Each iteration includes analysis, design, implementation and testing
  • Feedback is gathered after each iteration and used to guide the next
  • The final product emerges gradually rather than all at once
✅
Advantage: Problems are caught early, requirements can change between iterations, and the client can see progress regularly.

Agile Methodology

Agile is a popular family of iterative methodologies that prioritise flexibility, collaboration and delivering working software quickly. Key features:

  • Short development cycles called sprints (typically 1–4 weeks)
  • Close collaboration between the development team and client
  • Requirements can change throughout the project
  • Working software is delivered at the end of each sprint
  • Daily stand-up meetings to track progress

Waterfall Model

The waterfall model is a linear, sequential approach where each stage must be fully completed before the next begins. It was one of the earliest formal methodologies.

StageOutput
1. RequirementsRequirements specification document
2. DesignSystem and software design documents
3. ImplementationWorking code modules
4. TestingTest results and bug fixes
5. DeploymentReleased product
6. MaintenanceUpdates and patches

Comparing Methodologies

FeatureWaterfallIterative / Agile
ApproachLinear, one passCyclical, repeated passes
FlexibilityLow — changes are costlyHigh — adapts to change
Client involvementAt start and end onlyContinuous throughout
Risk of failureHigher — problems found lateLower — problems caught early
Best forFixed, well-defined requirementsEvolving requirements, complex projects
