HTML Document Structure

Every HTML5 page follows a standard structure:

Standard HTML5 Document
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Page Title</title>
  <link rel="stylesheet" href="style.css" />
</head>
<body>

  <!-- Page content goes here -->

  <script src="script.js"></script>
</body>
</html>

Semantic Elements

Semantic elements have meaningful names that describe the role of their content — making HTML more readable for developers, screen readers, and search engines. At Higher level, six key semantic elements are required.

ElementPurposeTypical position
<header> Introductory content — typically logo, site title and main navigation Top of page or section
<nav> A set of navigation links (menus, breadcrumbs) Inside header or as a sidebar
<section> A thematic grouping of content with its own heading Main page area, grouped by topic
<article> Self-contained, independently distributable content (a blog post, news article) Inside <section> or main content
<aside> Content tangentially related to main content (sidebars, related links, adverts) Beside main content
<footer> Bottom of page/section — copyright, contact links, site map Bottom of page or section
Typical Page Using Semantic Elements
<body>
  <header>
    <h1>Sports Club Website</h1>
    <nav>
      <a href="index.html">Home</a>
      <a href="events.html">Events</a>
      <a href="contact.html">Contact</a>
    </nav>
  </header>

  <main>
    <section>
      <h2>Upcoming Events</h2>
      <article>
        <h3>Summer Tournament</h3>
        <p>Join us on 15th July...</p>
      </article>
    </section>

    <aside>
      <h2>Recent News</h2>
      <p>Club AGM minutes published...</p>
    </aside>
  </main>

  <footer>
    <p>&copy; 2025 Sports Club</p>
  </footer>
</body>
💡
Non-semantic alternatives like <div> and <span> are still used (especially for CSS styling purposes), but semantic elements are preferred for page structure because they convey meaning to browsers, assistive technologies and search engines.

Forms

HTML forms allow users to input data. They are used for registration, login, search, feedback and ordering. Every form needs an opening and closing <form> tag, and a submit button to send the data.

Basic Form Structure
<form action="process.php" method="post">

  <label for="username">Username:</label>
  <input type="text" id="username" name="username" />

  <input type="submit" value="Log In" />

</form>

Key <form> attributes:

  • action — the URL where form data is sent when submitted
  • method — get (data in URL) or post (data in request body)

Input Types

The <input> element uses the type attribute to create different kinds of form controls. The following are required at Higher level:

TypeElement / AttributeExample use
Text <input type="text"> Single-line text input: name, username, postcode
Radio button <input type="radio"> Select one option from a group: gender, payment method
Checkbox <input type="checkbox"> Select zero or more options: interests, dietary requirements
Drop-down list <select> <option> Choose from a list: country, year group
Submit button <input type="submit"> Send the form data
Textarea <textarea></textarea> Multi-line text: message, comments
Form with Multiple Input Types
<form action="register.php" method="post">

  <!-- Text input -->
  <label for="name">Full Name:</label>
  <input type="text" id="name" name="name" required />

  <!-- Radio buttons (only one can be selected) -->
  <p>Gender:</p>
  <input type="radio" id="male" name="gender" value="male" />
  <label for="male">Male</label>
  <input type="radio" id="female" name="gender" value="female" />
  <label for="female">Female</label>

  <!-- Checkboxes (multiple can be selected) -->
  <p>Interests:</p>
  <input type="checkbox" id="tennis" name="sport" value="tennis" />
  <label for="tennis">Tennis</label>
  <input type="checkbox" id="swimming" name="sport" value="swimming" />
  <label for="swimming">Swimming</label>

  <!-- Drop-down list -->
  <label for="country">Country:</label>
  <select id="country" name="country">
    <option value="scot">Scotland</option>
    <option value="eng">England</option>
    <option value="wal">Wales</option>
  </select>

  <!-- Textarea -->
  <label for="message">Message:</label>
  <textarea id="message" name="message" rows="4" cols="40"></textarea>

  <!-- Submit button -->
  <input type="submit" value="Register" />

</form>
ℹ️
Radio buttons in the same group must share the same name attribute — this tells the browser they are mutually exclusive. Each radio button should have a unique id to link it to its <label>.

Tables

HTML tables organise data into rows and columns. They should be used for tabular data, not for page layout (CSS handles layout).

HTML Table Structure
<table>
  <thead>
    <tr>
      <th>Name</th>
      <th>Event</th>
      <th>Score</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Alice Smith</td>
      <td>100m Sprint</td>
      <td>12.4s</td>
    </tr>
    <tr>
      <td>Bob Jones</td>
      <td>Long Jump</td>
      <td>5.8m</td>
    </tr>
  </tbody>
</table>
TagPurpose
<table>Wraps the entire table
<thead>Groups the header row(s)
<tbody>Groups the data rows
<tr>A table row
<th>A header cell (bold, centred by default)
<td>A data cell
colspan="n"Attribute — makes a cell span n columns
rowspan="n"Attribute — makes a cell span n rows
